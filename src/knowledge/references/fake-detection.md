# 假懂检测

"假懂"是学习中最隐蔽的效率杀手——用户真的以为自己懂了，但一到检验就暴露。此模块提供系统化的假懂识别和检测方法。

## 假懂的10种信号

| # | 信号 | 典型表现 | 检测方法 |
|---|------|---------|---------|
| 1 | 无法复述 | 合上书想不起核心内容 | 闭书白纸默写，遗漏率>30%即为假懂 |
| 2 | 术语障碍 | 只能用术语解释术语，无法用日常语言 | "用中学生能听懂的话再讲一次" |
| 3 | 类比失效 | 能给出类比但说不出类比哪里不适用 | "这个类比的边界在哪里？什么情况下这个类比是错的？" |
| 4 | 变式卡壳 | 会做原题但变个条件就做不出来 | 改变一个非本质条件（数值/场景），看能否正确识别 |
| 5 | 无法预测 | 不能预测参数变化后的结果走向 | "如果把这个参数加倍，结果会怎么变？" |
| 6 | 回避原因 | 被问"为什么"时绕圈子 | 连续追问3次"为什么"，观察是否最终到达原子事实 |
| 7 | 过度自信 | 对答案过于确定但实际错误率高 | 做5题后先自评信心分 (1-5)，再对答案看差距 |
| 8 | 方法混乱 | 不知道该用什么方法，凭感觉选 | "这道题为什么用A方法而不是B？" |
| 9 | 边界模糊 | 不知道概念的适用条件和失效场景 | "在什么情况下这个结论不成立？" |
| 10 | 知识孤岛 | 单个概念能讲，但说不清和其他概念的关联 | "这个概念和你上周学的X有什么关系？" |

## 检测协议

### 快速检测（3分钟，单概念）

1. 合上材料，用一句话说出这个概念的**核心思想**（不允许用术语）
2. 举一个**非教科书上的例子**来说明
3. 说出这个概念的**一个常见误解**，以及为什么是误解

**判定**：3问全部流畅回答 → 可能真懂（需深度检测确认）。任何一问卡壳 → 存在假懂成分。

### 深度检测（10分钟，单概念+应用）

1. **定义层**：闭书默写核心公式/定义
2. **理解层**：用自己的话解释"为什么是这样"
3. **结构层**（最关键）：将5个相关概念和5个不相关概念混在一起，让用户按深层原理分类（而非表面特征）
4. **迁移层**：给一个改变条件后的变式题
5. **元认知层**：让用户预测"如果把条件X改成Y，结论会发生什么变化"，然后预测自己的准确度

**判定**：结构层正确率<80%或元认知预测偏差>20% → 知识结构有问题，即使定义层和理解层正确。

### 批量检测（跨多个知识点）

1. 列出所有知识点的名称
2. 让用户画概念关系图（连线标注关系类型：因果/层级/类比/对立）
3. 检查遗漏的连接和错误的连接

**判定**：遗漏≥30%的已知连接 → 知识网络不完整。存在≥2条错误连接 → 存在系统性误解。

## 不同知识类型的假懂特征

| 知识类型 | 最常见的假懂形式 | 针对性检测 |
|---------|----------------|----------|
| 事实性知识 | 记住了但无法在需要时提取 | 间隔后的突然测试 |
| 概念性知识 | 能说定义但不能迁移 | 变式题 + 分类任务 |
| 程序性知识 | 能跟着做但不能独立做 | 不提供任何提示，白板独立完成 |
| 元认知知识 | 不知道自己的薄弱点在哪 | 信心-准确度校准测试 |

## 假懂处理的对话模板

### 发现假懂时

```
"我注意到一个有意思的现象：你刚才的解释在[某个环节]和标准理解有偏差。
这不是'记错了'，而是一个常见的迷思——[解释迷思的本质]。

让我们重新梳理一下：先从最简单的直觉开始...[重新构建理解]

现在再来试一次：[同一个概念的新角度提问]"
```

### 用户回答模棱两可时

```
"你的回答方向是对的，但我需要确认你到底是真理解了还是模糊感觉到了方向。
用一个具体的例子来说明——不是教材上的例子，是你自己想的或者经历过的。"
```

### 用户反复卡在同一概念时

```
"这已经是我们第三次讨论这个概念了。让我换一个角度——
我们先不看书上的定义。想想看，如果没有这个概念，[领域]会缺少什么？
它在解决什么实际问题时是不可替代的？"
```

## 预防假懂的教学策略

1. **预判式教学**：新概念讲解前先预警"80%的学生在X这里会想错"
2. **即时验证**：每个概念讲解后，立刻给一个检测问题（不等到"学完再说"）
3. **随机抽查**：在后续学习中随机突击提问之前的内容（非预排期的突然检测）
4. **反向思维训练**："这个定理在什么情况下不成立？构造一个反例"
5. **输出驱动**：要求用户写教程/做讲解——写作是最诚实的检验
