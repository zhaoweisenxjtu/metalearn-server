# 实践编排

认识（knowing）和实践（doing）是两件事。实践编排的目标是将理解转化为可稳定输出的能力。

## 实践编排原则

- **难度梯度**：模板题→变式题→综合题→陷阱题→开放题
- **覆盖全面**：每个核心知识点都有对应练习，且不只有"会做题"还有"会选择方法"
- **间隔与交叉**：核心知识点在后续练习中以不同形式复现；不同类型题目交叉出现
- **限时与节奏**：标注建议用时，培养考试/实战节奏

## 方法选择决策树

根据目标类型和当前层级选择实践方法：

```
实践编排起点
│
├─ 目标类型：考试型
│   ├─ 当前 L1-L2 → 模板题 + 主动回忆（先确保会做标准题）
│   ├─ 当前 L3 → 变式题 + 交错练习（强化迁移，避免题型固化）
│   └─ 当前 L4-L5 → 陷阱题 + 限时模拟（查漏补缺，时间压力适应）
│
├─ 目标类型：应用型
│   ├─ 当前 L1-L2 → 跟做项目 + 组块化（复制完整流程，建立肌肉记忆）
│   ├─ 当前 L3 → 半自主项目 + 精细追问（给定框架，填充关键逻辑）
│   └─ 当前 L4-L5 → 开放项目 + 代码审查（端到端独立完成，同行评审）
│
└─ 目标类型：兴趣型
    ├─ 广度优先 → 引文网络 + 费曼技巧（抓领域骨架）
    └─ 深度优先 → 深度研究 + 多AI交叉质询（逐层深挖）
```

## 三种实践路径

### 路径A：考试型实践

适用于：学校考试、资格认证、标准化测试

| 阶段 | 内容 | 占比 | 方法 |
|------|------|------|------|
| 基础巩固 | 按知识点做模板题，建立题型-解法映射 | 30% | 组块化 + 主动回忆 |
| 交叉提升 | 混合题型随机练习，打破题型识别依赖 | 30% | 交错练习 + 限时 |
| 模拟实战 | 完整模拟考试，时间压力 + 真实环境 | 20% | 限时模拟 + 错题归因 |
| 查漏补缺 | 针对薄弱考点专项突破 | 20% | 精细追问 + 费曼技巧 |

检验标准：模拟考得分达到目标的 90%+，且能在不同题型间灵活切换方法。

### 路径B：应用型实践

适用于：编程项目、工程设计、业务实施

| 阶段 | 内容 | 时间占比 | 方法 |
|------|------|---------|------|
| 模仿阶段 | 跟随教程完整复现一个项目 | 25% | 组块化 + 双重编码 |
| 改编阶段 | 基于模仿的项目做功能扩展 | 25% | 精细追问 + 主动回忆 |
| 半自主 | 给定需求和框架，填充核心实现 | 25% | 交错练习 + 费曼技巧 |
| 全自主 | 从零设计并实现端到端项目 | 25% | 多AI交叉质询 + 代码审查 |

检验标准：能独立完成一个中等复杂度的端到端项目，遇到错误能自主调试。

### 路径C：混合型

适用于：既有考试需求又有应用需求的场景（如大学编程课）

将考试型和应用型按 4:6 比例混合。考试型练习覆盖理论考点，应用型项目加深理解和迁移。

| 阶段 | 理论侧（40%） | 实践侧（60%） |
|------|-------------|-------------|
| 入门 | 概念 + 标准题 | 跟做项目 |
| 进阶 | 变式题 + 交错练习 | 半自主项目 |
| 精通 | 模拟考 + 陷阱题 | 开放项目 |

## 练习设计清单

设计每个知识点的练习时检查以下维度：

- [ ] 有标准应用（模板题）—— 确保 L3 达标
- [ ] 有变式（改条件但不改原理）—— 区分 L3 和 L4
- [ ] 有陷阱（嵌入常见迷思概念）—— 检测假懂
- [ ] 有交叉（和至少一个其他知识点联合）—— 检验知识网络
- [ ] 有开放题（无唯一答案，考察方法选择）—— 检测 L5 元认知
- [ ] 标注了建议用时 —— 培养节奏
