# 学习目标诊断

在开始教学前，先诊断用户的学习目标、现状和约束，确保后续方法选择和计划编排匹配实际需求。

## 诊断流程

### 第一步：目标类型分类

向用户问清以下核心问题，确定目标类型：

| 维度 | 诊断问题 | 分类选项 |
|------|---------|---------|
| 目标性质 | 学完后需要做什么？ | 通过考试 / 实际做项目 / 满足好奇心 |
| 时间预算 | 有多少时间投入？ | <1天突击 / 1周集中 / 1月系统 / 长期持续 |
| 当前水平 | 对这个领域了解多少？ | 零基础 / 有一些碎片知识 / 较系统的入门 / 有经验需深化 |
| 材料情况 | 有什么学习材料？ | 教材/课程 / 论文 / 项目文档 / 无，需从头找 |

### 第二步：目标类型判定

| 目标类型 | 标志特征 | 核心需求 | 侧重点 |
|---------|---------|---------|--------|
| **考试型** | 有明确考纲、题型、时限 | 覆盖考点、快速解题、时间压力适应 | 检验（高频考点）+ 实践（限时训练） |
| **应用型** | 需要做出可运行的成果 | 端到端能力、真实约束、调试技能 | 实践（项目驱动）+ 教学（原理理解） |
| **兴趣型** | 无硬性时限、追求广度或深度 | 建立心智模型、激发持续兴趣 | 教学（直觉优先）+ 深度研究 |

### 第三步：前置知识检测

对于选定的学习主题，列出必要的前置概念清单，逐一快速检测：

1. 列出学习目标涉及的所有前置概念
2. 对每个前置概念，让用户用一句话解释
3. 根据解释质量判定：已掌握 / 部分掌握 / 未掌握
4. 对"未掌握"的前置概念，决定是补课还是绕过

检测模板：
```
学习 [目标] 需要以下前置知识，逐一确认当前理解程度：

| 前置概念 | 一句话解释（用户填写） | 状态 |
|---------|---------------------|------|
| [概念A] |                     |      |
| [概念B] |                     |      |
```

### 第四步：输出学习计划

综合以上信息，输出结构化学习计划：

```
## 学习计划：[主题]

**目标类型**：[考试型/应用型/兴趣型]
**时间预算**：[X天/周/月]
**当前层级**：[L1-L5]

### 推荐方法组合
- 主要方法：[根据目标类型推荐]
- 辅助方法：[根据内容特点推荐]
- 复习策略：[间隔重复排期建议]

### 阶段划分

| 阶段 | 内容 | 方法 | 预计用时 | 检验标准 |
|------|------|------|---------|---------|
| 1. 建立直觉 |       |      |         |         |
| 2. 形式化理解 |       |      |         |         |
| 3. 练习巩固 |       |      |         |         |
| 4. 综合迁移 |       |      |         |         |

### 风险预警
- [可能遇到的难点]
- [常见卡壳点]
```

## 目标类型 → 方法映射

| 目标类型 | 首选方法 | 教学方法侧重 | 检验方法侧重 | 实践路径 |
|---------|---------|------------|------------|---------|
| 考试型 | 交错练习 + 主动回忆 | 考点→原理逆向讲解 | 限时模拟、变式题 | 考试型路径 |
| 应用型 | 费曼技巧 + 组块化 | 原理→实践正向讲解 | 端到端项目、调试 | 应用型路径 |
| 兴趣型 | 深度研究 + 精细追问 | 直觉构建、知识网络 | 概念图、迁移解释 | 开放探索 |

## 计划调整信号

教学中出现以下情况时回到诊断环节重新评估：

- 用户反复卡在同一概念上 → 前置知识检测遗漏
- 练习正确率持续 <50% → 难度梯度过陡
- 用户表示"太简单" → 当前层级判断偏低
- 用户失去兴趣 → 目标类型可能判断错误（应转为兴趣型或应用型）
