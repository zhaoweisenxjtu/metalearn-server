# 精细追问（Elaborative Interrogation）

## 原理

学习时不断追问"为什么"，强迫大脑在已有知识和新信息之间建立因果连接。孤立的事实容易被遗忘，但嵌入因果网络的事实有多条提取路径——一条路断了还有其他路。

**核心机制**：精细加工（elaborative encoding）——不是"把信息存进去"，而是"把信息和已有知识编织在一起"。追问"为什么"和"如果...会怎样"是最简单的精细加工方式。

## 适用场景

- 学习任何有因果逻辑的陈述性知识
- 阅读教材、论文、文档时的主动阅读策略
- 弥补被动阅读的不足——把"浏览"变成"对话"

## 局限性

- 对无因果关系的纯事实（如专有名词、日期）效果有限
- 如果自己对问题给出了错误答案且不验证，可能强化错误
- 连续追问可能偏离主线——需要有意识地锚定核心问题

## 实操步骤

1. **阅读中暂停**：每读完一个段落或核心段落，停下来
2. **问三个为什么**：这个概念为什么是这样？它和前面学的东西有什么关系？如果某个条件变了会怎样？
3. **尝试自己回答**：先不看材料，用自己的知识尝试回答——即使不完全对
4. **验证答案**：对照材料或搜索确认答案是否正确
5. **记录洞见**：把追问发现的新连接记下来

## 组合建议

- **+费曼技巧**：费曼讲解中发现的模糊点，用精细追问深挖
- **+主动回忆**：主动回忆失败的知识点，回溯追问"当时为什么没理解"
- **+引文网络**：阅读论文时追问每篇文献"为什么这篇被反复引用"

## 对话示例

> **教练**："你刚读完了关于'市场失灵'的章节。为什么外部性会导致市场失灵？"
>
> **用户**："因为生产者只考虑自己的成本，不考虑对社会的外部成本..."
>
> **教练**："好。那如果政府不对污染征税，市场自己有没有办法解决这个问题？"
>
> **用户**："嗯...科斯定理说如果交易成本为零，双方可以谈判解决..."
>
> **教练**："那为什么现实中污染问题还是这么严重？交易成本为零这个假设在什么情况下成立？这就是你下一步需要追问的方向。"
