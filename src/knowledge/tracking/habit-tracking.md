# 习惯追踪

学习不是一次性事件，而是持续的习惯。本模块提供自包含的学习习惯追踪协议，无需外部工具依赖。

## 每日学习节奏

| 时段 | 时长 | 方法 | 记录方式 |
|------|------|------|---------|
| 专注时段 | 25-50min | 番茄钟 + 主动回忆 | 学习日志记录 |
| 发散时段 | 5-10min | 散步/放松/回顾 | — |
| 复习时段 | 15min | 间隔重复 | SM-2 排期系统 |
| 沉淀时段 | 10min | 记录今日学习事件 | learning-journal.json |

## 日志记录协议

### 每日学习日志

每天学习结束后，记录以下内容到 `~/.meta-learning/learning-journal.json`（格式见 `references/data-persistence.md`）：

1. **投入时间**：专注学习总时长（分钟）
2. **学习主题**：今天学了什么（关键词列表）
3. **使用方法**：用了哪些方法（费曼/主动回忆/交错练习等）
4. **亮点**：今天最清晰的理解时刻
5. **卡点**：仍然模糊或困难的点
6. **明日计划**：明天重点学什么

## 每周复盘

每周日进行 15 分钟复盘，对照学习日志和 SM-2 统计数据。

### 复盘清单

1. **层级进展**：本周有多少知识点从 L1-L2 升到了 L3+？
2. **薄弱点扫描**：SM-2 中评分持续 <3 的项目有哪些？是否需要换方法？
3. **方法有效性**：本周哪种方法带来的"清晰时刻"最多？
4. **时间审计**：实际投入 vs 计划投入对比，偏差原因？
5. **下周调整**：基于以上数据调整学习计划和排期

```

|         |         |         |      |

### 薄弱点
| 知识节点 | SM-2评分 | 卡在哪里 | 对策 |
|---------|---------|---------|------|
|         |         |         |      |

### 时间审计
```

## 知识沉淀路径

```
每日学习日志（learning-journal.json）
    ↓ 每周聚合
每周复盘
    ↓ 模式识别
月/季度回顾
    ↓ 结构化输出
可分享的学习笔记 / 教程 / 知识图谱
```

## 习惯养成梯度

| 阶段 | 时长 | 目标 | 核心习惯 |
|------|------|------|---------|
| 启动期 | 第1周 | 建立记录习惯 | 每天记录学习日志，写"卡点" |
| 适应期 | 第2-4周 | 建立每日节奏 | 固定时间学习+固定时间记录 |
| 巩固期 | 第2-3月 | 建立每周复盘 | 周复盘雷打不动，基于数据调整 |
| 自动化 | 第4月+ | 方法自动匹配 | 看到主题就知道用什么方法 |

## 中断恢复协议

如果中断超过3天：

1. **不追进度**：过去的日志不补，从今天开始重新记录
2. **减量重启**：首次回到原计划的50%强度
3. **SM-2 积压处理**：`python sm2-scheduler.py schedule` 查看积压列表，每天额外复习3项
4. **根因记录**：在学习日志中记录中断原因，识别模式

## 数据隐私

- 所有数据存储在本地 `~/.meta-learning/`，不自动上传
- 数据格式为纯 JSON 和 Markdown，可被任何文本编辑器读取
- 备份：建议每周导出一次 `tar -czf backup.tar.gz ~/.meta-learning/`
